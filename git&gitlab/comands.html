Comandos de Git y Gitlab
git init » Inicia el seguimiento de los cambios en una rama de archivos.
git status » Muestra el estatus de los archivos nuevos o modificados en memoria de "Staging".
git add » Agrega los archivos nuevos o modificados a la memoria de "Staging" (memoria chache), algunos de sus parametros pueden ser: "archivo.txt", ., -A, -am (Agregar todos).
git commit » Agrega los archivos a la memoria del disco duro, asignandole un identificador al/los archivo(s), usa como parametro -m + "descripcion" para asignar una descripcion a los cambios realizados, de lo contrario se ejecutara el editor de texto de consola VIM.
git log » Muestra todo el historial de cambios en la rama especifica (HEAD).
git show » Muestra el texto del archivo que ha sido modificado.
git branch » Muestra todas las ramas de trabajo y donde se encuentra ubicado.
git rm --cached + "archivo" » Elimina el archivo de la memoria de staying(RAM) antes de ser comentado.
